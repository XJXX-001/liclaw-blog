# Finishing a Development Branch

## 触发条件

所有实施任务完成，准备收尾时使用。

## 最终检查

### 1. 完整功能验证

```
[ ] 从头到尾走一遍用户流程
[ ] 所有主要路径测试过
[ ] 边界条件测试过
```

### 2. 测试最终确认

```bash
# 运行所有测试
[test command]

# 确认全部通过
# 如果有失败 → 修复后再继续
```

### 3. 代码审查

```
[ ] 没有明显 bug
[ ] 没有调试代码残留
[ ] 没有 TODO/FIXME 遗漏
[ ] 命名清晰
[ ] 代码结构合理
```

### 4. 文档同步

```
[ ] README 更新（如需要）
[ ] API 文档更新（如需要）
[ ] CHANGELOG 更新（如需要）
```

## 提交最终更新

```bash
# 确认所有更改已提交
git status

# 如有未提交更改，评估是否应该提交
```

## 合并准备

### 检查合并冲突

```bash
# 检查与目标分支的差异
git fetch origin
git diff main...HEAD

# 如果有冲突，准备解决
```

### 描述变更

准备清晰的变更描述：

```markdown
## 变更摘要

### 新增
- 功能 A：描述
- 功能 B：描述

### 修改
- 行为变化：描述

### 修复
- bug 修复：描述

### breaking changes
（如有）
```

## 提交给人类审查

向人类汇报：

```
所有实施任务完成。最终检查：

✅ 功能验证 — 通过
✅ 测试 — 全部通过 (X 个)
✅ 代码质量 — 通过
✅ 文档 — 已更新

准备就绪，请审查后合并。

变更摘要：
[简要描述]
```

## 禁止事项

- ❌ 不验证就提交
- ❌ 跳过代码审查
- ❌ 合并前不告诉人类
- ❌ 遗留调试代码
- ❌ 大量未提交更改

## 何时停止

如果发现问题：
1. 小问题 → 直接修复
2. 大问题 → 创建新任务，重新执行
